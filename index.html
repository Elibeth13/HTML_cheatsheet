    HTML STRUCTURE ANATOMY SECTIONS FUNDAMENTALS

What is HTML?

HTML (HyperText Markup Language) is used to give content to a web page and instructs web browsers on how to structure that content.

HTML structure

HTML is organized into a hierarchy, we can compare HTML elements with a family tree structure.
When we nest one HTML tag into another one, the tag that is nested inside is called the child element of the prior tag.
An HTML document is constructed with a lot of element members, so HTML elements can have parents, grandparents, siblings, children, grandchildren, etc.

Anatomy of an HTML element

An HTML element uses the following syntax: opening tag + content + closing tag.

<p> This is the content between the tags <p>

Anatomy of an HTML document

Individual HTML elements combine to form an entire HTML page.

Tag Name Description
<!DOCTYPE html> Document Type Declaration The first line of an HTML document
<html> HTML Element The root element after the !DOCTYPE declaration. All content for an HTML document should be contained between the opening and closing tags
<head> Head Element Contains metadata and includes the title of the HTML document and links to stylesheets.
<title> Title Element Defines the title of the page, which is the title that appears in the browser tab the page is loaded in.
<body> Body Element This contains all the content of an HTML document.

Basic sections of an HTML document

Webpages tend to share similar standard components. HTML boasts a number of block level elements used to define areas of a website.
In the HTML code, we can mark up sections of content based on their functionality.
Structured content makes the reading experience easier and more enjoyable:

  • Users looking at a web page tend to scan quickly to find relevant content.
  • Without headings, a page will perform poorly in terms of SEO (Search Engine Optimization).
  • Visually impaired people can use assistive technologies like screen readers to recognize those elements and help with tasks like "find the main navigation", or "find the main content."
  • To style content so CSS and JavaScript can effectively target it.
Tag Name Description
<header> header Represents a group of introductory content
<nav> navigation bar Contains the main navigation functionality for the page
<main> main content Directly inside <body>. It is for content unique to the page
<aside> sidebar Often placed inside <main>. It contains content that is not directly related to the main content
<footer> footer Represents a group of end content for a page

HTML fundamental elements

Tag Name Description
<!-- and closing --> comment Content inside of comments will not be rendered by browsers
<a> anchor element Creates a link to another page or to a location in the current page
<b> bring attention to element Used to draw attention to a section of text, usually rendered in boldface
<blockquote> block quotation element A section of a document which contains a longer quotation, usually spanning multiple lines
<br> line break element produces a line break in text (carriage-return)
<button> button element Represents a button meant to be clicked by the user
<cite> citation element It is used to mark up the title of a cited creative work
<code> inline code element Displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code
<del> deleted text element Shows text that is to be removed from a document, usually rendered in strike-through text
<div> content division element It is the generic container for flow content
<em> emphasis element Represents text which is emphasized. Browsers will show the enclosed text in italics
<embed> embed external content element Represents text which is emphasized. Browsers will show the enclosed text in italics
<figcaption> figure caption element represents a caption or legend describing the rest of the contents of its parent <figure>element
<figure> figure element Represents self-contained content, potentially with an optional caption, which is specified using the <figcaption> element.
The figure, its caption, and its contents are referenced as a single unit.
<form> form element Represents a document section containing interactive controls for submitting information
<h1>-<h6> heading elements Represent six levels of section headings,with <h1> being the highest level of heading and <h6> being the lowest
<img> image element Displays an image on the web page
<input> input (form input) element It is used to create interactive controls for web-based forms in order to accept data from the user
<li> list item element Represents a single item in a list of items.
It must be contained in a parent element: an ordered list, an unordered list, or a menu tag
<link> external resource link element Connects the current page with an external file
<menu> menu element Represents an unordered list of items with more semantic meaning than a regular ul element
<ol> ordered list element Represents an ordered list of items
<option> option element Represents one option in a dropdown created by the select tag
<p> paragraph element Contains and displays a block of text that defines a paragraph
<select> select element Creates a drop-down list for the user to select from a number of option elements
<source> media or image source element Represents an interface for adding source content to the page
<span> content span element Used for grouping related text or elements for styling and scripting
<strong> strong importance element Used to identify text that is very important, or urgent
<style> style information element Applies CSS styles to an HTML document
<ul> unordered list element Represents an unordered list of items
<video> video embed element Represents an interface for adding video content to the page
